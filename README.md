# 泊展科技标号管理系统

这是一个面向上海泊展科技水下清洗机器人业务的网页版标号管理系统原型。系统按企业级项目结构设计，当前版本使用原生 Node.js + 原生前端实现，不依赖外网安装包。

## 启动

```powershell
cd E:\WorkSpace\产品\标号管理\numbering-system
node server.mjs
```

浏览器打开：

```text
http://localhost:4173
```

## 已实现功能

- 按编号大类和分类字段自动生成编号。
- 支持 PD、GP、SP、TL、IG、MT、BL、PK、TR、TD、SW 共 11 类规则。
- 支持编号台账查询、分页、搜索、状态筛选。
- 支持新增、修改、删除编号记录。
- 支持分类代码维护，新增分类后立即参与编号生成。
- 支持上传图纸、照片、PDF、STEP、CAD 等附件。
- 支持 CSV / JSON 导出。
- 后端统一生成流水号，避免多人使用时重复编号。
- 支持账号登录、角色权限、后端接口强制鉴权。
- 支持评审提交、管理员批准受控、受控记录锁定、ECN 变更单。
- 支持产品 SN 追溯、BOM 父子编号关联。
- 支持管理员查看近期修改日志。
- 支持个人工作区：员工只能看到自己的草稿/评审中/驳回记录，以及全公司共享的受控记录。
- 支持管理员待审批队列：集中处理员工提交的新增申请和修订申请。
- 支持受控文件修订申请：员工不能直接改受控文件，只能复制出个人修订稿，修改后提交审批，管理员批准后生成新版本，旧版本自动转为历史版本。

## 默认账号与权限

当前版本取消固定五角色登录逻辑，改为“固定测试账号 + 登录时多选权限”。登录页不需要输入账号密码，系统固定使用 `admin / 123`，你只需要勾选本次登录要测试的权限，进入系统后即按这组权限显示和限制功能。

| 账号 | 密码 | 默认权限 |
| --- | --- | --- |
| `admin` | `123` | 固定测试账号，实际权限由登录页勾选决定 |
| `engineer` | `eng123` | 新建编号、修改草稿/退回记录、附件、提交评审、创建修订申请 |
| `reviewer` | `rev123` | 只读查看，可用于验证无编辑/无创建界面 |
| `viewer` | `view123` | 只读查看 |

细分权限包括：查看编号、新建编号、修改草稿/退回记录、上传/删除附件、提交审批、创建修订申请、查看待审批、审批通过、审批退回、删除记录、分类维护、账户管理、查看修改日志、系统配置、ECN/SN/BOM 管理等。权限由前后端共同控制：前端只显示当前账号有权限的功能；后端 API 也会强制校验，不能通过直接请求绕过审批、分类维护、编辑或删除等权限。

## 受控与 ECN 规则

- 工程师创建记录后可提交评审。
- 审批通过后，记录状态变为“受控”，并进入锁定状态。
- 任何账号都不能直接修改待审批或受控记录及附件，即使拥有全部权限也必须走审批/修订流程。
- 如需修改受控记录，应在详情页点击“创建修订申请”。系统会生成同编号的新版本修订稿，例如原始版本 `A0` 修订后生成 `A1`。
- 员工修改修订稿后提交审批，有审批权限的账号在“待审批”页面批准后，修订稿会成为新的最新版受控记录；原版本自动转为“历史版本”，不可修改，只能删除。
- 同一编号的所有版本保持关联，台账可看到 `A0`、`A1`、`A2` 等版本链；只有最新版受控记录可以继续创建修订申请或 ECN。
- ECN 仍可用于记录较正式的工程变更背景，但当前核心修改流程以“修订申请”为主。

## 个人可见性规则

| 记录状态 | 员工本人 | 其他员工 | 管理员 |
| --- | --- | --- | --- |
| 草稿 | 可见、可编辑 | 不可见 | 可见 |
| 评审中 | 可见、不可编辑 | 不可见 | 在待审批中可见 |
| 驳回 | 可见、可编辑 | 不可见 | 可见 |
| 受控 | 全公司可见、不可直接编辑 | 全公司可见、不可直接编辑 | 可见、可管理 |
| 修订申请 | 创建人可见、可编辑/提交 | 不可见 | 在待审批中可见 |

## 修改日志

管理员登录后可看到“修改日志”页面，系统会记录近期操作，例如：

- 登录
- 新建编号
- 修改记录
- 删除记录
- 分类新增/修改/删除
- 提交评审
- 批准/驳回
- 创建 ECN
- 创建 SN
- 创建 BOM 关联

## 当前存储方式

当前原型使用：

- `data/db.json`：规则配置、编号记录、流水号、审计记录。
- `data/uploads/`：上传附件。

正式部署时建议替换为：

- PostgreSQL / MySQL：编号记录、分类配置、审批记录。
- OSS / S3 / MinIO：图纸、照片、STEP、PDF 等大文件。
- Redis 或数据库事务：高并发流水号生成锁。
- 企业微信、飞书或 OIDC：企业登录和权限。

## 核心 API

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| GET | `/api/config` | 获取编号规则和分类配置 |
| PUT | `/api/config` | 整体更新编号配置 |
| POST | `/api/groups/:group` | 添加分类项 |
| PUT | `/api/groups/:group/:code` | 修改分类项 |
| DELETE | `/api/groups/:group/:code` | 删除分类项 |
| GET | `/api/records` | 分页查询编号记录 |
| POST | `/api/records` | 自动生成编号并创建记录 |
| GET | `/api/records/:id` | 查看记录详情 |
| PUT | `/api/records/:id` | 修改记录 |
| DELETE | `/api/records/:id` | 删除记录 |
| POST | `/api/records/:id/attachments` | 上传附件 |
| DELETE | `/api/records/:id/attachments/:attachmentId` | 删除附件 |
| GET | `/api/export.csv` | 导出 CSV |
| GET | `/api/export.json` | 导出 JSON |
